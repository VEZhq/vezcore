# Security policy

## Reporting a vulnerability

Do not report security vulnerabilities in public issues, pull requests, or discussions.

Use GitHub's private vulnerability reporting for this repository:

https://github.com/VEZhq/vezcore/security/advisories/new

Include the affected component, reproduction steps, impact, and any suggested mitigation. Do not include production credentials, personal data, or customer data. We will acknowledge a complete report as soon as possible and coordinate remediation and disclosure privately.

## Supported versions

Security fixes are applied to the current `main` branch. The `develop` branch may contain unreleased changes and must not be treated as production-supported.

## Scope

Reports concerning VEZcore application code, authentication, authorization, data access, deployment configuration, or bundled dependencies are in scope. Reports about third-party services should be sent to the relevant provider unless VEZcore's integration is the source of the issue.
